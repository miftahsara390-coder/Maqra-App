Project context
After learning how to consume a web API and use the phone's native sensors, you take a new step: building a complete application by adopting a modern professional workflow — AI-assisted "vibe coding".

A young cultural startup based in Casablanca wants to promote reading in Morocco. She entrusts you with the development of Maqra, a mobile application that transforms the smartphone into an intelligent personal library. The user catalogs his books — whether they come from the Casablanca Book Fair (SIEL), a bookstore in Rabat or the family library — follows his progress, sets reading objectives and visualizes his statistics. All 100% local, without any external web APIs.


The application must reflect Moroccan cultural wealth: support for books in arabic, French and Amazigh (Tamazight), a visual identity inspired by zellige and local colors (Majorelle blue, terracotta, mint green).

The particularity of this project is not only technical: it is a new way of working. You will generate the interface (UI) with it Google Stitch, without writing the logic yourself initially and then implementing it manually all business logic — using AI as guide and co-pilot, never as a crutch.

Educational objective
Adopt a "vibe coding" workflow: clearly separate the generation of UI (Stitch) from the implementation of logic (manual, guided by AI)
Use Google Stitch to generate quality React Native displays, then integrate and adapt the generated code
Manually implement all business logic: state, calculations, persistence — without copying and pasting AI-generated logic
Learn to use AI as a guide: ask the right questions, understand the answers, decide on the implementation yourself
Manage the project professionally with Jira (via MCP): cut into tickets, monitor progress, document
Respect the best architectural structure of a React Native Expo project (UI /logic /state separation)
Persist data locally with AsyncStorage and leverage the device camera
Manage multilingual content (RTL Arabic, French, Amazigh) — a real challenge for the Moroccan market
⚡ The rules of Vibe Coding
Read this section carefully: it defines what is allowed and what is not. This project is based on strict discipline.
✅ For UI — AI allowed (without logic)
You generate the screens with Google Stitch (https://stitch.withgoogle.com)
Stitch produces the design and presentation code: layout, visual components, styles
You integrate this code into your Expo project and adapt it (colors, spacing, props)
The generated UI must contain NO business logic (no stat calculation, no persistence, no complex state management)
🛠️ For logic — 100% manual implementation
All business logic is handwritten by you: Zustand store, progress calculations, statistics, persistence, camera permissions management
You DO NOT copy blocks of logic generated by AI
AI can be used as a GUIDE: How to calculate a percentage of progress? ", "What is the correct way to structure a Zustand blind? ", "Explain to me how to manage Arabic RTL in React Native"
You must understand and know how to explain every line of logic you write
⚠️ Key distinction to remember
UI = generated by AI (Stitch), without logic.
LOGIC = written manually, AI is only used to explain and guide.
In the oral defense, you will have to justify each logical decision. Logic code copied without understanding will be punished.
Application content
A 3-screen application that catalogs and tracks the user's reading, 100% locally:
Screen 1 — Library (Dashboard)
List of books: FlatList of all books with cover, title, author, language (ar/fr/amazigh) and status (to read /in progress /finished).
Annual objective: Animated progression ring: books read /goal of the year.
Quick stats: Total number of books, pages read, books completed this month.
Filters/search: Filter by status or language, search by title or author
Add button: Add a new book (manual entry + cover photo)
Screen 2 — Detail & Reading session
Book details: Cover, title, author, number of pages, language, status, rating (stars).
Progress: Bar /animated ring: current page /total number of pages.
Update: Enter the page reached → recalculated and animated progress
Reading stopwatch: Start /Pause /Stop a session with haptic feedback
Animated counters: Pages read and reading times that increment smoothly.
Change of status: Mark as "in progress" or "completed"
Screen 3 — Statistics & Profile
Profile picture: Taken with the device camera (expo-camera /expo-image-picker).
Cumulative stats: Total books, pages, reading time, series of days (streak)
History: Previous reading sessions (FlatList) with appearance animation
Simple graphic: Completed books per month (animated bars, without heavy bookstores).
Management: Delete a book or session.
Guiding principle
Components in /components contain only display (props in, JSX out).
All logic lives in /store, /lib and /hooks — and you write it.
